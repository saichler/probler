🎯 Why Choose Probler?

Probler addresses the fundamental pain points of building scalable, maintainable microservices architectures for network operations.

🔄

Model-Agnostic Services

Services that adapt to any data structure without requiring code changes, making your infrastructure future-proof.

🌐

Virtual Networking

Seamless service-to-service communication through intelligent virtual network overlay with built-in service discovery.

🛡️

Built-in Security

Comprehensive security layer with authentication, authorization, and built-in monitoring for all network operations.

📈

Horizontal Scaling

Scale collectors and parsers horizontally based on device count and data processing volume automatically.

🔧

Platform Agnostic

Deploy on bare metal, Docker, or Kubernetes with the same configuration and management interface.

📊

Real-time Monitoring

Interactive NOC dashboard with topology visualization, alarm management, and performance metrics.

🚀 Quick Start

1

Clone Repository

Get the latest version of Probler from GitHub

# Clone the repository git clone https://github.com/saichler/probler.git cd probler
2

Build Services

Build all microservices using the provided script

# Build all services cd go && ./build-images.sh
3

Deploy Platform

Choose your deployment method

# Docker Compose (Development) docker-compose up -d # Kubernetes (Production) cd k8s && ./apply-all.sh
4

Access Dashboard

Open the Network Operations Center

# Local access open https://localhost:443/probler/ # Default credentials: admin / admin

Prerequisites

Go 1.23.8+ Docker & Docker Compose Kubernetes cluster PostgreSQL database
