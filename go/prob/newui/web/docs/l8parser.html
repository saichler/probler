Overview

L8Parser is a model-agnostic parsing engine. It processes raw data collected by l8collector -- SNMP walks, SSH command outputs, REST API responses, and kubectl queries -- and transforms it into structured protobuf objects using a configurable rule-based framework. Parsed objects are sent to l8inventory via PATCH operations.

Processing Flow

  1. Job Reception -- Parser receives completed collection jobs from l8collector
  2. Rule Application -- Applies parsing rules based on l8pollaris configuration
  3. Data Transformation -- Transforms raw data into structured protobuf objects via PropertyId injection
  4. Inventory Update -- Sends parsed objects to l8inventory

Vendor Support

15 vendors with automatic detection by sysOID enterprise prefix: Cisco, Juniper, Palo Alto, Fortinet, Arista, Nokia, Huawei, Dell, HPE, IBM, Check Point, SonicWall, D-Link, Extreme, NEC. Each vendor configuration polls for system information, interface monitoring, hardware inventory, and performance metrics.

NVIDIA GPU (Three-Protocol Stack)

Full GPU monitoring via SNMP (7 polls), SSH (5 polls), and REST (4 polls). Covers system info, per-GPU metrics, utilization, temperature, power, NVLink topology, and health.

Kubernetes

Resource monitoring via kubectl: Nodes, Pods, Deployments, StatefulSets, DaemonSets, Services, Namespaces, NetworkPolicies. Includes log collection and multi-cluster support.

Routing Protocols

OSPF and BGP via standard MIB-II OIDs (vendor-agnostic). VRF via SSH show vrf parsing for 9 vendor-specific formats.
