l8topology

Topology

Network topology modeling and visualization.

View on GitHub All Components

Overview

A high-performance network topology visualization and management system. Provides real-time network topology discovery, multiple layout algorithms, and GPU-accelerated WebGL rendering for enterprise-scale network graphs (30K+ nodes, 300K+ links).

Core Components

TopoService
Core topology service managing nodes, links, and locations with in-memory caching. Handles CRUD, viewport-filtered queries, layout execution, and location aggregation.
TopoListService
Registry for managing multiple topologies. Clients query this to discover available topology services.
ITopoDiscovery
Extensible discovery interface. Implementations convert external inventory systems into topology nodes and links.
Layer1 Discovery
Built-in physical topology discovery from network device inventory.
WorldCities
Thread-safe geographic lookup resolving city names to latitude/longitude coordinates.
WebGL Renderer
GPU-accelerated rendering with instanced nodes, batched links, camera controls, and color-based hit-detection picking.

Layout Algorithms

Location
Geographic coordinates via Robinson projection on a world map. Supports location aggregation for co-located nodes.
Hierarchical
Tree-like layout from root nodes. Levels assigned by longest path.
Circular
Concentric rings with inner nodes positioned tighter than outer ones.
Radial
Spoke layout radiating from central hub nodes. Multiple centers detected by connectivity.
Force-Directed
Physics simulation using Coulomb repulsion and Hooke attraction. Up to 300 iterations with velocity damping (0.85).

Visualization Features

Interactive pan, zoom, and selection on a world map
Device type icons: Router, Switch, Firewall, Load Balancer, Access Point, Server, Storage, Gateway
Color-coded link status (Up/Down/Partial) with directional arrows
Bidirectional link merging
Viewport culling -- server-side filtering to only visible elements
Location aggregation -- groups co-located nodes with aggregate counts

Data Types

Core protobuf type L8Topology contains maps of nodes, links, and locations. Queried via L8TopologyQuery which specifies layout algorithm and optional viewport bounds.

Node types
Generic, Switch, Router, Network Aggregation, Firewall, Load Balancer, Access Point, Server, Storage, Gateway
Link directions
AsideToZside, ZsideToAside, Bidirectional
Link statuses
Up, Down, Partial

Performance

30,000+ nodes, 300,000+ links
WebGL instanced drawing for 10-30x speedup over SVG
Server-side viewport culling reduces data transfer
Location aggregation reduces visual complexity
