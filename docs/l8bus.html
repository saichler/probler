Overview

The Layer 8 Network BUS provides a comprehensive virtual networking layer built on top of TCP connections, enabling distributed systems to communicate as if they were on a single network. Latest updates include enhanced service activation, crash prevention fixes, and distributed cache improvements.

Virtual Networking

Creates a virtual network layer that abstracts physical network topology, allowing seamless communication between distributed nodes with enhanced stability and activation.

Service Discovery

Automatic service registration and discovery with health monitoring, leader election capabilities, and improved service linking.

Robust Routing

Intelligent message routing with support for unicast, multicast, service-based routing patterns, and optimized batch processing.

Core Components

VNet (Virtual Network)

Location: vnet/VNet.go

Purpose: Central network switch that manages connections between nodes

  • TCP listener for incoming connections
  • Message routing and switching
  • Health monitoring integration
  • Service discovery
  • Connection management

VNic (Virtual Network Interface)

Location: vnic/VirtualNetworkInterface.go

Purpose: Network interface for individual nodes to connect to the overlay

  • Bidirectional communication (TX/RX)
  • Keep-alive mechanisms
  • Automatic reconnection
  • Service API integration

Protocol

Location: protocol/Protocol.go

Purpose: Message serialization/deserialization and protocol handling

  • Base64 encoded message data
  • Sequence number management
  • Message creation and parsing
  • Transaction state management

Network Topology

Hybrid Network Architecture

The overlay creates a sophisticated hybrid network topology with three distinct connection types:

Internal Connections

Direct connections between processes on the same machine for high-performance local communication.

External Connections

TCP connections to remote VNet switches enabling cross-network communication.

Service Routing

Messages routed based on service names and areas, providing intelligent service-aware networking.

Usage Examples

Creating a VNet (Network Switch)

vnet := NewVNet(resources) err := vnet.Start() if err != nil { log.Fatal("Failed to start VNet:", err) }

Initialize and start the central network switch that will manage all node connections and route messages.

Creating a VNic (Network Interface)

vnic := NewVirtualNetworkInterface(resources, nil) vnic.Start() // Wait for connection to be established vnic.WaitForConnection()

Create a virtual network interface that connects your application to the Layer8 overlay network.

Service Registration

// Services are automatically registered through // the health system and can be discovered by // other nodes in the network. healthService := &health.HealthService{} resources.Services().RegisterServiceHandlerType(healthService) resources.Services().Activate("health", "HealthService", 0, resources, vnic)

Register services that will be automatically discovered by other nodes in the network.

Configuration

The overlay network is configured through SysConfig which includes:

  • Local and remote UUIDs and aliases
  • VNet port configuration
  • Service definitions
  • Queue sizes and data limits
