What's New in v1.3.0

Latest features and improvements

Service Level Agreement Framework

Complete refactoring of service activation with new ServiceLevelAgreement structure. Simplified API with encapsulated configuration for service lifecycle management.

Dynamic Type Creation

New NewOf() method in IRegistry enables dynamic type instantiation at runtime, perfect for flexible plugin architectures and runtime extensibility.

Improved API Ergonomics

Variadic parameters for SetPrimaryKeys() and SetArgs() methods. More intuitive API design with cleaner, more readable code.

Powerful Features

Everything you need for building distributed systems at scale

Virtual Network Interface

Advanced VNic abstraction with multiple communication patterns: Unicast, Multicast, Round-robin, Proximity-based routing, and Leader selection for optimal message distribution.

Token Authentication

Secure token-based authentication system with validation, activation mechanisms, and comprehensive error handling for robust security operations.

Service Discovery

Built-in service registration and discovery with area-based routing, health monitoring, and intelligent endpoint selection for resilient systems.

Health Monitoring

Real-time system health tracking with Unix top-style output, CPU/memory usage statistics, message flow metrics, and comprehensive logging.

Transaction Management

Distributed transaction support with state tracking, commit/rollback operations, error handling, and comprehensive transaction lifecycle management.

Protocol Buffers

Comprehensive type definitions with multi-language support, efficient serialization, cross-platform compatibility, and extensible schema system.

Code Examples

See how easy it is to build distributed systems

Service Level Agreement (v1.3.0) 
// Create Service Level Agreement
sla := &ifs.ServiceLevelAgreement{}
sla.SetServiceName("user-service")
sla.SetServiceArea(1)
sla.SetStateful(true)
sla.SetTransactional(true)
sla.SetReplication(true)
sla.SetReplicationCount(3)

// Variadic parameters (NEW in v1.3.0)
sla.SetPrimaryKeys("userId", "email")
sla.SetArgs(dbConn, cacheLayer, logger)

// Simplified activation (NEW in v1.3.0)
handler, err := services.Activate(sla, vnic)
if err != nil {
    log.Fatal("Failed to activate:", err)
}
Dynamic Type Creation (v1.3.0) 
// Register types with the registry
registry := ifs.NewRegistry()
userType := &User{
    Name: "template",
    Role: "default",
}
registry.Register(userType)

// Dynamic type instantiation (NEW in v1.3.0)
newUser := registry.NewOf(userType).(*User)
newUser.Name = "John Doe"
newUser.Email = "john@example.com"

// Create multiple instances dynamically
for i := 0; i < 10; i++ {
    instance := registry.NewOf(userType)
    // Each instance is a fresh copy
    processUser(instance)
}

// Perfect for plugin architectures
plugin := registry.NewOf(pluginTemplate)
plugin.Initialize(config)
plugin.Execute()

Quick Start Guide

Get up and running in minutes with Layer 8 Types v1.3.0

⚡ Upgrading from v1.2.0? Check out the API Migration Guide for breaking changes and new features.

1. Clone Repository

git clone https://github.com/saichler/l8types
cd l8types

2. Install Dependencies

cd go
go mod download

3. Generate Bindings

cd proto
./make-bindings.sh

4. Run Tests

cd go
./test.sh  # Automated tests with coverage
# Or manually:
go test -v -cover ./...

5. Build Your App

import "github.com/saichler/l8types/go/ifs"

// v1.3.0 - Create SLA
sla := &ifs.ServiceLevelAgreement{}
sla.SetServiceName("my-service")
handler, _ := services.Activate(sla, vnic)

6. Deploy

docker build -t myapp .
docker run myapp
