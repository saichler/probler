Powerful Features

Built for modern network infrastructure with enterprise-grade capabilities

🌐

Multi-Protocol Support

Native support for SNMP v2c, SSH, Kubernetes, REST/RESTCONF, and GraphQL. Extensible architecture for adding custom protocols.

⚡

High-Performance Concurrency

1:1 host-to-goroutine mapping enables true parallel data collection. Handle 1,000-10,000 devices per instance with optimal resource utilization.

🎯

Service Level Agreements

Integrated SLA support for better service lifecycle management. CollectorService and ExecuteService now operate with enhanced reliability and monitoring.

🔧

Dynamic Target Management

New TargetService and TargetCenter for centralized device configuration. Support for L8PTarget and dynamic configuration updates at runtime.

🛡️

Enhanced SNMP Resilience

Advanced timeout handling with automatic net-snmp fallback. Configurable timeouts and connection resilience for enterprise reliability.

🔄

Distributed Job Execution

ExecuteService enables remote job distribution across cluster nodes. Automatic load balancing and failover for mission-critical operations.

🎯

Dynamic Parameter Substitution

Flexible command templating with $variable syntax. Runtime parameter replacement for adaptable data collection scripts.

🚀

Optimized Performance

Unified string handling with l8utils package reduces memory allocations. Enhanced garbage collection efficiency and improved throughput.

Supported Protocols

Comprehensive coverage for modern network infrastructure

SNMP v2c

  • Community-based authentication
  • Configurable timeout and retry settings
  • OID-based data collection with MIB support
  • Enhanced timeout protection with net-snmp fallback
  • Support for SNMP walks and gets

SSH

  • Username/password authentication
  • Command execution with prompt detection
  • Session management and connection pooling
  • Configurable prompts and timeouts
  • Enhanced error handling and logging

Kubernetes

  • kubectl-based data collection
  • Context-aware configuration
  • Base64-encoded kubeconfig support
  • Cluster resource monitoring
  • Dynamic parameter substitution ($variable)

REST/RESTCONF

  • HTTP/HTTPS-based API data collection
  • Multiple authentication methods (token, basic)
  • Support for all HTTP methods (GET, POST, PUT, PATCH, DELETE)
  • Flexible body and response handling
  • Certificate-based secure connections

GraphQL

  • GraphQL query execution
  • API key and token-based authentication
  • Flexible query structure support
  • Typed response handling with protobuf
  • HTTPS with certificate support

Quick Start

Get up and running in minutes

Installation & Setup
# Clone the repository git clone https://github.com/saichler/l8collector.git cd l8collector/go # Initialize Go modules go mod init GOPROXY=direct GOPRIVATE=github.com go mod tidy go mod vendor # Build the application go build ./... # Run tests with coverage chmod +x test.sh ./test.sh
Service Integration
// Service activation with SLA support collectorService := &CollectorService{} sla := ifs.NewServiceLevelAgreement(collectorService, "collector", serviceArea, false, nil) err := collectorService.Activate(sla, vnic) // Execute Service with SLA slaExec := ifs.NewServiceLevelAgreement(&ExecuteService{}, "exec", serviceArea, false, nil) slaExec.SetArgs(collectorService) vnic.Resources().Services().Activate(slaExec, vnic) // Target device configuration target := &l8tpollaris.L8PTarget{ TargetId: "target-001", Hosts: []*l8tpollaris.Host{ // Host configurations }, } // Start data collection response := targetService.Post(object.New(target), vnic)

Latest Updates

Recent improvements and enhancements (October 2025)

🎯

SLA Integration

Service Level Agreement support added to core services. CollectorService and ExecuteService now activate with SLA for improved service management, lifecycle control, and monitoring capabilities.

🔧

Target Management Refactoring

Introduced TargetService and TargetCenter for centralized target device management. Support for L8PTarget and L8PTargetList types with dynamic configuration updates.

🧪

Enhanced Testing Framework

Comprehensive test coverage for GraphQL and REST collectors. Updated test suite with SLA integration tests and improved mock services for better test reliability.

📈

Performance Optimizations

Continued string handling optimizations using l8utils package. Enhanced memory management and reduced garbage collection pressure for better throughput at scale.

🌐

Protocol Enhancements

REST/RESTCONF and GraphQL collectors with full-featured support. Enhanced SNMP resilience with net-snmp fallback and improved timeout handling across all protocols.

🛠️

Service Components

New service components: JobCadence for job scheduling management, StaticJobs for configuration, and BootSequence for service initialization with improved startup reliability.
