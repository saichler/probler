Overview

Layer 8 Model Agnostic Parsing (L8Parser) serves as the intelligent data transformation layer within the Layer8 network management ecosystem. This powerful model-agnostic parsing service processes raw data collected by l8collector, applying sophisticated rule-based parsing framework with advanced PropertyId injection and protocol buffer integration. With support for multiple protocols and vendors, L8Parser ensures data consistency and reliability across diverse network infrastructures.

Vendor Limitation Documentation

Understanding Device Limitations

The notsupported.md file provides crucial information about attributes that cannot be collected via SNMP/SSH for specific vendors and device types.

  • Network Vendors: Cisco, Juniper, Arista, Nokia, Huawei - routing tables, environmental data limitations
  • Security Vendors: Palo Alto, Fortinet - session monitoring, threat detection constraints
  • Server Vendors: Dell, HPE, IBM - hardware monitoring, IPMI integration limits
  • Protocol Limitations: SNMP v1/v2c/v3 constraints, MIB availability issues
  • Alternative Methods: NETCONF, REST APIs, Telemetry streaming options

Key Unsupported Areas

Real-time Metrics

Network health metrics requiring continuous monitoring such as jitter, packet loss percentage, and availability scores

Hardware Details

CPU architecture, memory frequency, and other hardware specifics not exposed via standard SNMP MIBs

Topology Discovery

Network-wide topology mapping requiring visibility across multiple devices simultaneously

Historical Data

Performance trends and historical metrics requiring time-series data collection and storage
