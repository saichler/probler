Overview

Layer 8 Model Agnostic Parsing (L8Parser) serves as the intelligent data transformation layer within the Layer8 network management ecosystem. This powerful model-agnostic parsing service processes raw data collected by l8collector, applying sophisticated rule-based parsing framework with advanced PropertyId injection and protocol buffer integration. With support for multiple protocols and vendors, L8Parser ensures data consistency and reliability across diverse network infrastructures.

Features

Advanced Rule-Based Parsing

Sophisticated parsing engine with pluggable rules including Contains, Set, StringToCTable, and CTableToMapProperty for comprehensive data transformation scenarios.

PropertyId Path Injection

Intelligent property path resolution with automatic slice index and map key injection using reflect.Kind values for complex nested data structures.

Multi-Vendor SNMP Support

Comprehensive SNMP support for 9 major vendors with automatic OID-based device detection and vendor-specific configurations optimized for hardware monitoring.

Protocol Buffer Integration

Seamless protobuf integration with field name mapping and type-safe data structure handling for consistent inventory representation.

Real-time Processing

Event-driven processing of collector jobs with concurrent rule execution and optimized memory management for high-performance data transformation.

Enhanced Error Handling

Comprehensive error handling with property resolution validation, parsing error recovery, and graceful degradation for robust operation.

PropertyId Injection System

Intelligent Path Resolution

The new PropertyId injection system automatically resolves complex nested paths by injecting slice indices and map keys. For example, networkdevice.physicals.interfaces.name becomes networkdevice.physicals<{24}physical-0>.interfaces<{2}0>.name for proper property resolution.

Usage

Enhanced Integration

import ( "github.com/saichler/l8parser/go/parser/service" "github.com/saichler/l8types/go/ifs" ) // Create parsing service with enhanced error handling parsingService := &service.ParsingService{} err := parsingService.Activate( "parsing-service", serviceArea, resources, vnic, &NetworkDevice{}, // Target protobuf object "Id", // Primary key field )

PropertyId Injection Example

// PropertyId with automatic injection for nested collections attr := &types.Attribute{ PropertyId: "networkdevice.physicals.interfaces.name", Rules: []*types.Rule{ { Name: "Set", Params: map[string]*types.Parameter{ "from": {Value: ".1.3.6.1.2.1.2.2.1.2.1"}, }, }, }, } // Automatically becomes: // "networkdevice.physicals<{24}physical-0>.interfaces<{2}0>.name"

Advanced SNMP Configuration

// Enhanced SNMP polling with corrected OID relationships poll := &types.Poll{ Name: "systemMib", What: ".1.3.6.1.2.1.1", // Walk OID (container) Operation: types.Operation_OMap, Attributes: []*types.Attribute{ { PropertyId: "networkdevice.equipmentinfo.sysname", Rules: []*types.Rule{ { Name: "Set", Params: map[string]*types.Parameter{ "from": {Value: ".1.3.6.1.2.1.1.5.0"}, // Specific leaf OID }, }, }, }, }, }
