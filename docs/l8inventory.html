Key Features

🔄

Generic & Model Agnostic

Works with any data model through Protocol Buffers definitions. Supports network devices, Kubernetes resources, and custom inventory types.

🌐

Distributed Architecture

Built on the Layer 8 ecosystem for distributed operations with enhanced service link support and robust caching infrastructure.

🔍

Performance Optimized Querying

Enhanced SQL-like query capabilities with significant performance improvements, advanced filtering, and optimized fetch operations.

⚡

Real-time Operations

Support for POST, PATCH operations with immediate notifications, service linking, and configurable primary key management.

🔗

Enhanced REST API

Complete web service interface with improved REST API endpoints and advanced integration capabilities for external systems.

📡

Advanced Forwarding

Enhanced forwarding support to downstream services with service link architecture for complex data processing pipelines.

📊

Built-in Statistics

Comprehensive statistics collection and monitoring capabilities for performance analysis and system health tracking.

🛠️

Filter Optimization

Recent filter improvements and fetch optimizations providing faster data retrieval and more efficient query processing.

Installation & Usage

Prerequisites

  • Go 1.23.8 or later
  • Layer 8 ecosystem dependencies

Quick Start

cd go/ ./test.sh # Manual setup go mod init GOPROXY=direct GOPRIVATE=github.com go mod tidy go mod vendor go test -tags=unit -v -coverpkg=./inv/... -coverprofile=cover.html ./...

Basic Usage

import ( inventory "github.com/saichler/l8inventory/go/inv/service" "github.com/saichler/l8types/go/ifs" ) // Activate inventory service serviceName := "inventory" serviceArea := byte(0) primaryKey := "id" elemType := &YourProtobufMessage{} vnic.Resources().Services().Activate( inventory.ServiceType, serviceName, serviceArea, resources, listener, primaryKey, elemType, )

Supported Operations

POST

Add new inventory items to the distributed cache with automatic validation and indexing.

PATCH

Update existing inventory items with delta changes and real-time notifications.

GET

Query and retrieve inventory data using SQL-like syntax with advanced filtering.

DELETE

Remove inventory items with proper cleanup and cascade operations support.

  • Logical interfaces

Kubernetes Resources
